# T3Series Enterprise 설치 가이드

이 문서는 T3Series Enterprise 설치에 대한 가이드를 제공하며, T3Series Enterprise의 모든 모듈이 동일한 Host에 설치된다는 전제로 가이드 문서를 기술하고 있습니다. 여러 Host에 설치되는 경우라면, 자이오넥스 연구소(rnd@zionex.com)로 모듈별 다른 Host에 설치하는 방법에 대해서 문의해 주시기 바랍니다.

T3Series Enterprise는 Demand Planning, Sales Replenishment Planning, Inventory Management, Replenishment Planning, Master Planning, Production Planning을 위한 여러 모듈을 제공해 주는 SCM 솔루션이며, 아래 그림과 같이 T3Series Enterprise Service Bus를 통해서 각각의 모듈간의 통신이 이루어지는 구조입니다.

# 1. 설치 전 준비 사항

# 설치 전 필수 소프트웨어

T3Series Enterprise 설치를 위해 다음과 같은 소프트웨어가 설치되어 있어야 합니다.

  1. JAVA 8 (필수)
  2. Web Application Server (필수)
  3. Database (필수)
  4. Python (권장)

설치 경로 및 서비스 이름은 자이오넥스 기술지원을 위해서 아래와 같이 표준 경로로 설정해주시기 바랍니다.

T3Series Enterprise는 윈도우의 경우 "C:\Zionex" 경로에 설치하고, 리눅스의 경우 "/opt/zionex" 경로에 설치합니다.

# JAVA 8

Open JDK 8 설치를 권장하며, "https://github.com/ojdkbuild/ojdkbuild"에서 다운로드 할 수 있습니다. JAVA 8 하위 또는 상위 버전 모두 지원하지 않으므로, JAVA 8 최신 버전을 설치해야 합니다.

JAVA 설치 후, 환경 변수 JAVA_HOME을 설정합니다.

JAVA 8이 설치된 Home 디렉토리에 대한 JAVA_HOME 시스템 환경 변수를 설정하고, PATH 시스템 환경 변수에 JAVA_HOME의 bin 디렉토리를 설정합니다.

  1. "제어판 > 고급 시스템 설정 > 환경변수"에서 설정이 가능합니다.

  2. JAVA_HOME 시스템 환경 변수를 추가합니다. (ex: JAVA_HOME=C:\Apps\java\java-1.8.0-openjdk-1.8.0.201-1)

  3. Path에 Java 실행 파일의 위치를 추가합니다. (ex: Path=%JAVA_HOME%\bin;...)

    • Java의 Path 설정은 최 상단에 설정해야 합니다. 이는 이미 설치된 Java 환경보다 우선순위를 높이기 위해서입니다.

# Web Application Server

Java Servlet 3.1 이상을 지원하는 무료/상용 Web Application Server를 설치합니다. Deploy는 각 솔루션 별로 차이가 있으므로 개별적으로 진행하셔야 합니다.

개발 환경은 무료로 제공되는 Web Application Server인 Apache Tomcat 9 버전을 권장하며, 기술 지원이 필요한 경우 상용 Web Application Server를 설치해야 합니다.

T3Series Enterprise 시스템에 적용된 Web Application Server 환경의 레퍼런스는 다음과 같습니다.

  • Apache Tomcat: S-Oil
  • JBoss: 현대건설기계
  • WebLogic: 두산전자
  • JEUS: 롯데칠성

Apache Tomcat 9는 https://tomcat.apache.org/download-90.cgi에서 다운로드 가능합니다.

윈도우의 경우, "C:\Tomcat 9" 경로에 설치하고, 서비스 이름은 "Tomcat9"로 설정합니다. 리눅스의 경우, "/opt/tomcat9" 경로에 설치하고, 서비스 이름은 "tomcat9"로 설정합니다.

# Database

Oracle 또는 SQL Server가 설치되어 있어야 합니다. Oracle은 11g 이상, SQL Server는 2012 이상 버전 설치를 권장합니다. 설치된 Host Address / Database Service Port / Database Name or Service Name / User ID / User Password 정보를 설치 전에 미리 확인해야 합니다.

# Python (권장)

Python 설치는 필수 사항은 아닙니다.

  • Python을 설치하지 않을 경우, T3Series Enterprise 설치를 각 모듈 별로 직접 설치를 해야 합니다. 또한, configuration_datasource.xml 설정도 각 모듈 별로 직접 설정해야 합니다.

  • Python을 설치 할 경우, T3Series Enterprise 자동 설치 및 Configuration 자동 설정을 할 수 있습니다. Python3을 설치해야 하며, https://www.python.org/downloads/ 에서 다운로드 할 수 있습니다.

  • 설치 시, "Add Python 3.7 to PATH" 체크 박스를 설정합니다.

# 리얼그리드 라이센스 발급

T3Series Enterprise Web Application이 사용하는 리얼그리드 라이센스 발급을 진행합니다.

  • 리얼그리드 라이센스 발급 절차는 자이오넥스 관리팀으로 요청해서 진행합니다. (프로젝트와 개발/운영을 구분해서 요청해야 합니다.)

  • 자이오넥스 관리팀으로 라이센스 발급을 요청하면, 해당 업체를 통해서 라이센스 발급을 위한 제품 번호를 부여받게 됩니다. 자이오넥스 관리팀은 제품 등록과 프로젝트 멤버에게 라이센스 관리를 위임하는 역할까지 수행합니다.

  • 생성된 프로젝트 정보는 다음과 같습니다.

  • 해당 프로젝트에 등록된 멤버와 등록된 제품 정보를 확인할 수 있습니다.

  • 등록된 제품 번호를 이용하여, 운영 또는 개발 라이센스를 발급 받을 수 있습니다.

    • 아래 이미지의 하단 버튼 3개 중 두번째 버튼이 운영 라이센스를 위한 버튼이고, 세번째 버튼이 개발 라이센스 발급을 위한 버튼입니다. 첫번째 버튼은 워터마크가 존재하는 라이센스 발급을 위한 것입니다.
    • 해당되는 라이센스 발급을 요청하고, 리얼그리드 업체의 승인이 완료되면 라이센스 및 제품을 다운로드 할 수 있습니다. 승인 처리는 업무 시간에만 진행되니 참고하시기 바랍니다.
    • 발급 요청시 해당 서버의 IP와 도메인 정보 그리고 로컬 호스트 정보를 콤마로 구분해서 모두 등록해야 합니다. 등록되지 않은 Host는 리얼그리드를 사용할 수 없습니다.

# 2. T3Series Enterprise 샘플 데이터 파일 설정

자이오넥스 연구소에서는 샘플 데이터 파일을 제공하고 있습니다. 샘플 데이터가 필요한 경우, 연구소(rnd@zionex.com)로 요청합니다.

다음은 SQL Server와 Oracle 샘플 데이터를 적용하는 방법입니다.

# SQL Server

  1. 원본 데이터베이스에서 전체 백업을 진행합니다.

    BACKUP DATABASE T3SERIES
TO DISK = 'C:\t3series-enterprise-data-sqlserver.bak'
   WITH FORMAT,
      MEDIANAME = 'T3Series Enterprise',
      NAME = 'Full Backup of T3Series Enterprise'
GO

  2. 전체 백업된 샘플 데이터 파일을 복사하고, 확인합니다.

    # 아래 경로에 백업 파일을 복사합니다.
C:\t3series-enterprise-data-sqlserver.bak

  3. 아래 쿼리로 논리적/물리적 이름을 확인합니다.

    RESTORE FILELISTONLY FROM DISK='C:\t3series-enterprise-data-sqlserver.bak'
GO

  4. 논리적/물리적 이름이 확인된 "T3SERIES"가 아닌 "T3SAMPLE"로 쿼리를 설정하면 아래와 같이 에러가 발생합니다. (주의)

    RESTORE DATABASE T3SAMPLE
FROM DISK='C:\t3series-enterprise-data-sqlserver.bak'
   WITH MOVE 'T3SAMPLE' TO 'C:\Program Files\Microsoft SQL Server\MSSQL11.MSSQLSERVER\MSSQL\DATA\T3SAMPLE.mdf',
      MOVE 'T3SAMPLE_log' TO 'C:\Program Files\Microsoft SQL Server\MSSQL11.MSSQLSERVER\MSSQL\DATA\T3SAMPLE_log.ldf'
GO

  5. 아래 쿼리로 새로운 데이터베이스("T3SAMPLE")에 복원합니다.

    RESTORE DATABASE T3SAMPLE
FROM DISK='C:\t3series-enterprise-data-sqlserver.bak'
   WITH MOVE 'T3SERIES' TO 'C:\Program Files\Microsoft SQL Server\MSSQL11.MSSQLSERVER\MSSQL\DATA\T3SAMPLE.mdf',
      MOVE 'T3SERIES_log' to 'C:\Program Files\Microsoft SQL Server\MSSQL11.MSSQLSERVER\MSSQL\DATA\T3SAMPLE_log.ldf'
GO

# Oracle

  1. 원본 데이터베이스에서 전체 백업을 진행합니다.

    C:\> exp userid=T3SERIES/T3SERIES file=C:\t3series-enterprise-data-oracle.dmp owner=T3SERIES

  2. 전체 백업된 샘플 데이터 파일을 복사하고, 확인합니다.

    # 아래 경로에 백업 파일을 복사합니다.
C:\t3series-enterprise-data-oracle.dmp

  3. 사용자 테이블스페이스 및 계정을 생성합니다.

    C:\> sqlplus "/as sysdba"

-- DROP TABLESPACE T3SAMPLE INCLUDING CONTENTS AND DATAFILES CASCADE CONSTRAINTS;
CREATE TABLESPACE T3SAMPLE DATAFILE 'C:\Apps\oracle\oradata\orcl\T3SAMPLE.DBF' SIZE 3G;

-- DROP USER T3SAMPLE CASCADE;
CREATE USER T3SAMPLE IDENTIFIED BY T3SAMPLE
   DEFAULT TABLESPACE T3SAMPLE
   TEMPORARY TABLESPACE TEMP;

GRANT CONNECT, RESOURCE, DBA TO T3SAMPLE;

  4. 새로 생성된 계정에 데이터를 복원합니다.

    C:\> imp userid=T3SAMPLE/T3SAMPLE file=C:\t3series-enterprise-data-oracle.dmp fromuser=T3SERIES touser=T3SAMPLE

# 3. T3Series Enterprise 설치 파일 다운로드

필요한 모듈의 설치 파일을 자이오넥스 연구소(rnd@zionex.com)로 요청합니다.

Service Bus, Data Server, Web Application(Wing UI)은 필수 설치 파일 입니다.

  1. T3Series Service Bus (필수)
  2. T3Series Data Server (필수)
  3. T3Series Web Application (필수)
  4. T3Series Demand Plan Server (선택)
  5. T3Series Sales Replenishment Plan Server (선택)
  6. T3Series Master Plan Server (선택)

# 4. T3Series Enterprise 설치

해당 가이드는 T3Series Enterprise 의 전체적인 설치 흐름에 대한 설명을 중심으로 작성되었습니다. 자세한 설명은 전달된 T3Series Enterprise 메뉴얼을 참고하시기 바랍니다.

t3series-servicebus, t3series-dataserver, t3series-wingui는 필수 설치 파일 입니다.

# 1. 설치 경로에 설치 파일을 복사합니다.

  • T3Series Enterprise 릴리즈 파일인 t3series-servicebus1.0.zip, t3series-dataserver1.0.zip, t3series-demandplanserver1.0.zip, t3series-srpserver1.0.zip, t3series-masterplanserver1.0.zip, t3series-wingui1.0.zip 파일을 복사합니다.

  • Python이 설치 되어 있고, 자동 설치를 할 경우, t3series.py 파일을 복사합니다.

# 2. 발급 받은 리얼그리드 라이센스를 리얼그리드 경로에 복사합니다.

리얼그리드 라이센스는 개발/운영 서버에 맞게 적용되어야 합니다. 리얼그리드 라이센스가 없다면 그리드를 사용할 수 없습니다.

설치 경로 아래에 Web Application Server에 Deploy 할 WAR 파일과 동일한 이름의 폴더를 생성합니다.

# t3series.war 폴더 생성
C:\Zionex\t3series.war

해당 폴더에 적용된 파일은 위치한 경로 및 하위 경로에서 동일한 이름의 파일을 탐색해서 업데이트하게 됩니다.

리얼그리드 라이센스 업데이트를 위해서 아래와 같이 발급 받은 라이센스 파일을 복사합니다.

# 해당 경로에 발급 받은 리얼그리드 라이센스를 복사합니다.
c:\Zionex\t3series.war\vendors\realgridjs

# 3. T3Series Enterprise를 설치합니다.

T3Series Enterprise는 각 모듈을 직접 실행하는 방법과 OS의 서비스로 등록하여 실행하는 방법이 있습니다.

아래는 OS 서비스로 등록하는 방법에 대한 설명입니다.

Python이 설치 된 경우, t3series.py 파일를 사용하여 자동 서비스 등록 및 Configuration 설정이 가능합니다. Python 설치가 되어있지 않은 경우, 사용 할 모듈의 Configuration 설정 및 서비스 등록을 직접 적용해야 합니다.

Python이 설치 된 경우

t3series.py 파일을 실행하면 자동으로 압축 파일이 해제되며, 이미 압축이 해제된 상태라면 해제하지 않습니다. 압축이 해제된 이후에는 Data Source 정보를 입력하게 되며, 이후 Configuartion 설정 및 서비스 등록은 자동으로 적용됩니다.

  1. Window 환경의 경우, 관리자 모드로 Command 창을 실행하고, Linux 환경의 경우 터미널 창을 실행합니다. (Linux 환경의 경우, sudo 권한이 필요합니다.)

  2. 설치 파일이 위치한 경로로 이동하여, python t3series.py install 명령어를 실행합니다. (Linux 환경의 경우, python3 명령어로 실행합니다.) 해당 명령어는 압축 파일을 해제하고, Configuration 설정 및 WAR 파일 업데이트, 서비스 등록, 시작까지 적용됩니다.

  3. Data Source에 대한 설정 정보를 입력합니다. Data Source 정보는 한번 입력으로 모든 모듈의 configuration_datasource.xml에 자동 적용됩니다. configuration_datasource.xml 파일을 직접 수정하지 않아도 되며, WAR 파일에 포함된 configuration_datasource.xml 파일도 자동 적용됩니다. 만약, Data Source 정보가 모듈 별로 차이가 있다면 직접 수정해야 합니다.

Python을 설치하지 않은 경우

사용 할 모든 설치 파일을 압축 해제 한 후, 각 모듈 별 Configuration 설정과 서비스 등록 또는 서버 구동을 진행합니다.

  1. 설치 하고자 하는 모듈의 압축 파일을 해제합니다. t3series-servicebus, t3series-dataserver, t3series-wingui는 필수 모듈이므로 설치 항목에 꼭 포함시킵니다.

  2. 각 모듈의 configs 디렉토리 아래에 있는 configuration_datasource.xml 설정을 합니다.

    # 모듈 별 configuration_datasource.xml 경로
C:\Zionex\t3series-dataserver1.0\configs\configuration_datasource.xml
C:\Zionex\t3series-demandplanserver1.0\configs\configuration_datasource.xml
C:\Zionex\t3series-masterplanserver1.0\configs\configuration_datasource.xml
C:\Zionex\t3series-srpserver1.0\configs\configuration_datasource.xml

    t3series-servicebus는 configs 디렉토리 아래에 있는 auth-server / notice-server / workflow-server의 configuration_datasource.xml 파일을 추가로 설정합니다.

    # auth-server
C:\Zionex\t3series-servicebus1.0\configs\auth-server\configuration_datasource.xml

# notice-server
C:\Zionex\t3series-servicebus1.0\configs\notice-server\configuration_datasource.xml

# workflow-server
C:\Zionex\t3series-servicebus1.0\configs\workflow-server\configuration_datasource.xml

    아래 Configuraion 설정은 일반적인 configuration_datasource.xml 설정에 대한 예제입니다.

    주의사항은 t3series-demandplanserver, t3series-masterplanserver 모듈이 고정된 Data Source ID를 사용하고 있다는 점을 주의하셔야 합니다. t3series-demandplanserver 모듈은 H2 DB를 사용하며, Data Source ID가 "H2_DP_OUTPUT"로 설정되어야 합니다. URL 설정도 "jdbc:h2:file:C:/Zionex/t3series-demandplanserver1.0/configs/dpoutput;AUTO_SERVER=TRUE"와 같이 절대 경로를 사용해야 합니다. t3series-masterplanserver 모듈은 계획 수립을 위한 Engine은 Data Source ID가 "ENGINE"으로 적용되어야 하며, T3Series Enterprise UI 서비스는 Data Source ID가 "UI"로 적용되어야 합니다. 그리고, Engine의 Data Source Type은 "MSSQL_2_YN" / "ORACLE_2"가 아닌 "MSSQL" / "ORACLE"로 적용되어야 합니다. 이러한 다양한 케이스를 보완하기 위해서 자동화된 설치 방법을 권장합니다.

    # configuration_datasource.xml 설정 예시
<t3series>
  <data-sources default="[default data-source id]">
    <data-source id="[data-source id]" name="[data-source name]">
      <type>[database type : MSSQL_2_YN or ORACLE_2]</type>
      <url>[type 별 url 설정]</url>
      <user encryption="false">[DB USER]</user>
      <password encryption="false">[DB PASSWORD]</password>

  </data-sources>
</t3series>

ORACLE_2: <url>jdbc:oracle:thin:@[Host Name]:[Port]:[Database Name]</url>
MSSQL_2_YN: <url>jdbc:jtds:sqlserver://[Host Name][:Port]/[Database Name]</url>

<t3series>
  <data-sources default="T3SERIES">
    <data-source id="T3SERIES" name="T3SERIES">
      <type>MSSQL_2_YN</type>

</t3series>

# 7. 서버 및 포트 이슈 관련 확인 방법

T3Series Enterprise 설치 시, 포트 확인 및 방화벽 문제 등으로 인해 포트가 비활성화 되는 경우가 많습니다. 아래와 같은 방법으로 서버의 Host 접속 테스트 및 Service 접속 테스트를 통해 포트 활성화 여부를 확인할 수 있습니다.

# 1. SQL Server에서 기본 포트인 1433 정적 포트로 적용하는 방법

SQL Server의 경우, 기본 설치 시 동적 포트로 적용됩니다. 기본 포트인 1433 정적 포트로 적용해야 합니다.

  1. SQL Server 구성 관리자(SQL Server Configuration Manager)에 접속합니다. 좌측 트리 메뉴에서 SQLEXPRESS에 대한 프로토콜을 선택하고, 우측 화면에서 TCP/IP를 선택합니다.

  2. TCP/IP 속성 창에서 IP 주소 탭을 선택합니다. 하단의 "IPALL" 옵션에서 "TCP 동적 포트" 값을 삭제하고, "TCP 포트"에 "1433"으로 설정합니다.

  3. 적용 후, SQL Server를 다시 시작 합니다.

# 2. SQL Server가 "SQL Server 인증 모드" 및 "Windows 인증 모드" 모두 활성화되어 있는지 확인하는 방법

SQL Server의 "Windows 인증 모드"는 커넥션 설정 방식이 다르므로, "Windows 인증 모드 & SQL Server 인증 모드"로 변경해야 합니다.

  1. 서버에서 "Windows 인증"으로 로그인 한 후, SQL Server 오른쪽 클릭 후 "속성"을 선택합니다.

  2. 서버 속성 창에서 "보안" 탭을 선택한 후, "서버 인증"에서 "SQL Server 및 Windows 인증 모드"를 선택합니다.

인증모드 관련 참고

https://docs.microsoft.com/ko-kr/sql/relational-databases/security/choose-an-authentication-mode?view=sql-server-2017

# 3. 방화벽에 포트 활성화시키는 방법

사용 할 포트가 활성화 되지 않으면 연결 오류가 발생할 수 있습니다. 방화벽에서 사용 할 포트를 활성화 시키는 방법 입니다.

  1. 제어판에서 "시스템 및 보안"을 선택합니다. Windows Defender 방화벽을 선택합니다.

  2. 좌측의 "고급 설정"을 선택합니다.

  3. "고급 보안이 포함된 Windows Defender 방화벽" 창에서 좌측의 "인바운드 규칙" 선택 후, 우측의 "새 규칙"을 선택합니다.

  4. "포트" 선택 후, 좌측의 "프로토콜 및 포트"를 확인합니다.

  5. "TCP"/"UDP" 선택 후, "특정 로컬 포트"에 활성화 할 포트 번호를 입력합니다.

  6. "연결 허용"에 선택합니다.

  7. "도메인", "개인", "공용"에 선택합니다.

  8. 규칙에 사용할 이름과 설명을 정의합니다.

  9. "인바운드 규칙" 탭에서 정의한 규칙을 확인합니다.

# 6. 다른 Host에서 Telnet 명령어로 해당 Host의 서비스에 접속 테스트

다른 Host에서 해당 Host의 서비스로 접근이 가능한지 Telnet 명령어로 접속 테스트를 합니다.

윈도우 10에는 기본으로 Telnet 명령을 사용할 수 없습니다. 제어판에서 해당 기능을 활성화하여 사용해야 합니다. 혹은 command 창에서 telnet 입력 시 에러가 발생한다면, 아래 방법으로 해당 기능을 활성화 해야 합니다.

  1. 제어판에서 프로그램을 선택합니다.

  2. "프로그램 및 기능"에서 Windows 기능 켜기/끄기를 선택합니다.

  3. Window 기능 창에서 텔넷 클라이언트 를 체크한 후, 확인 버튼을 선택합니다.

Telnet 으로 Post connection 테스트 방법

  • Command 창에서 "telnet ip_address port"를 입력합니다.

    # telnet 테스트
C:\Zionex>telnet 192.168.0.31 1433

화면이 바뀌고 커서만 깜빡거리거나, 서비스 목록이 나온다면 해당 서버에 포트가 정상적으로 서비스되고 있는 상태입니다.

# 7. Java 메모리 설정

각 서버의 Java Heap Size를 설정하여 사용 할 최대 메모리 크기를 적용할 수 있습니다. 메모리 설정은 엔진 성능에 영향을 주는 요소이므로 매우 중요합니다.

  1. server.cmd server.cmd 파일을 사용하는 경우, bin 디렉토리 아래에 있는 server.cmd 파일을 문서 편집기를 이용하여 설정을 변경합니다.

    • Xmx 옵션: 최대 Heap Size (최대로 사용할 수 있는 메모리 크기)
    java -Xmx4g

  2. wrapper.conf

    서비스로 등록하여 사용하는 경우, "\wrapper\conf" 안에 있는 wrapper.conf 파일을 문서 편집기를 이용하여 설정을 변경합니다.

    • wrapper.java.maxmemory: 최대 Heap Size (최대로 사용할 수 있는 메모리 크기)
    # Maximum Java Heap Size (in MB)
# alternative for wrapper.java.additional.<n>=-Xmx
wrapper.java.maxmemory=1024

# 8. Python 설치 명령어 정리

  • python t3series.py license: "t3series-"로 시작되는 T3series Enterprise 배포 파일 기준으로 t3series.license 경로에 license.xml 파일을 생성합니다.
  • python t3series.py setting: 배포된 압축 파일이 해제되어 있지 않으면, 압축을 해제하고 데이터소스 설정 및 war 파일 업데이트까지 적용합니다.
  • python t3series.py install: setting 명령어가 수행하는 작업을 포함해서, 서비스 등록 및 서비스 시작까지 적용합니다.
  • python t3series.py stop: 서비스가 설치된 역순으로 서비스를 종료합니다.
  • python t3series.py start: 서비스가 설치된 순서로 서비스를 시작합니다.
  • python t3series.py restart: stop 명령어를 수행하고, start 명령어를 수행합니다.
  • python t3series.py uninstall: 설치된 서비스를 종료하고, 서비스를 제거하며, 설치된 파일까지 제거합니다.
